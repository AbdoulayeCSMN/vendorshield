Avilyre predicts **operational risk** per supplier — **delivery-delay** and **defect
(PPM)** probability — from delivery history.

## Prediction model

**Delay probability** — logistic regression over features computed on an expanding window
(no temporal leakage):

$$
P(\text{delay}) = \sigma(z), \qquad z = \mathbf{w}^{\top}\mathbf{x}, \qquad \sigma(z) = \frac{1}{1 + e^{-z}}
$$

where $\mathbf{x}$ gathers the bias, prior OTD, prior average delay, quantity, PPM and
seasonality ($\sin$/$\cos$). Weights $\mathbf{w}$ are learned by **gradient descent**
minimizing the log-likelihood with $L_2$ regularization:

$$
\mathcal{L}(\mathbf{w}) = -\frac{1}{n}\sum_{i=1}^{n}\Big[y_i\log \hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\Big] + \lambda\lVert \mathbf{w}\rVert^{2}
$$

A factor's **contribution** $j$ (the driver shown to the user) is $c_j = w_j \cdot x_j$.

**PPM threshold breach** — from the forecast PPM $\widehat{\text{PPM}}$ (linear trend) and the
threshold $\tau$ (default $5000$):

$$
P(\text{breach}) = \sigma\!\left(1.7 \cdot \frac{\widehat{\text{PPM}} - \tau}{\sigma_{\text{res}}}\right)
$$

where $\sigma_{\text{res}}$ is the standard deviation of the trend residuals.

## Cold-start: value on day one

<Note>
A new account has no history. Instead of failing, Avilyre uses an **anonymized global
model** (aggregate cross-account coefficients, **no tenant identifier exposed**) to produce
an initial prediction (capped confidence). As soon as the account has enough data, a
dedicated model takes over automatically.
</Note>

## When models train

| Trigger | Scope |
|---|---|
| "Recompute prediction" button (supplier record) | One supplier |
| "Recompute predictions" button (suppliers list) | The whole account |
| Cron `POST/GET /api/ml/retrain` | All accounts |
| New account with no history | Global model (cold-start) |

Batch retraining is **fast and free** (no LLM call, deterministic explanation). See
[Automation](/en/deployment/automation) for scheduling.

## Privacy

The global model only learns **aggregate coefficients**; it neither stores nor exposes any
named data from another customer. Each account only sees its own predictions (RLS).

## Financial distress

A **bankruptcy risk** prediction is available per supplier (dedicated tab on the record),
powered by the `bankruptcy-predictor` Edge Function.
